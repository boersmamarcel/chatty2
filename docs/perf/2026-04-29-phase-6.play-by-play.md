# Phase 6 play-by-play

1. All new motion reads `subtle_guidance::use_reduced_motion()`.
2. Caret, whisperer dot, composer strip, skeleton shimmer, and stream smoothing each have inline fallback comments.
3. Reduced motion preserves geometry and disables time-based smoothing.
4. Whisperer text uses theme muted foreground and skeleton shimmer uses low-opacity muted foreground to stay theme-derived.
5. Composer focus is not moved during waiting; the existing GPUI input entity remains focused.
6. ARIA-specific requirements are documented as a platform divergence because GPUI does not expose DOM ARIA attributes.

Synthetic metrics: desktop GPUI has no ARIA/CLS/browser a11y harness in-repo. The code centralizes reduced-motion behavior and avoids adding focus transfers.
